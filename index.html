การเชื่อมต่อ Front-end และ Back-end เป็นกระบวนการที่ทำให้แอปพลิเคชันที่สร้างขึ้นบนฝั่งผู้ใช้ (Front-end) สามารถสื่อสารและแลกเปลี่ยนข้อมูลกับฝั่งเซิร์ฟเวอร์ (Back-end) ซึ่งทำให้แอปพลิเคชันสามารถทำงานได้อย่างสมบูรณ์แบบ

กระบวนการหลักในการเชื่อมต่อมีดังนี้:

- การส่งคำขอ (Request) จาก Front-end ไปยัง Back-end

- การประมวลผลข้อมูลใน Back-end:

- การรับข้อมูลจาก Back-end ใน Front-end

  • สาระการเรียนรู้
  • จุดประสงค์การเรียนรู้
  • สมรรถนะประจำหน่วย

1.ความเข้าใจในกระบวนการทำงานของเว็บไซต์หรือแอปพลิเคชันทั้งระบบ

2.การสื่อสารระหว่าง Front-end และ Back-end ผ่าน API (Application Programming Interface)

3.ความรู้เกี่ยวกับการใช้ HTTP request/response (GET, POST, PUT, DELETE)

4.การจัดการข้อมูลในฐานข้อมูลและการแสดงผลใน Front-end

5.การใช้เครื่องมือและเทคโนโลยีต่างๆ เช่น JavaScript, Fetch API, AJAX, JSON, RESTful API

6.ความเข้าใจเกี่ยวกับการรักษาความปลอดภัยในการเชื่อมต่อระหว่าง Front-end และ Back-end

1. นักเรียนสามารถอธิบายกระบวนการเชื่อมต่อระหว่าง Front-end และ Back-end ได้

2. นักเรียนสามารถสร้างคำขอ (Request) จาก Front-end และรับข้อมูล (Response) จาก Back-end ผ่าน API

3. นักเรียนสามารถใช้เครื่องมือเช่น Fetch API, Axios หรือ AJAX เพื่อเชื่อมต่อกับเซิร์ฟเวอร์

4. นักเรียนสามารถทำความเข้าใจและใช้งาน RESTful API ได้

5. นักเรียนสามารถแสดงผลข้อมูลที่ได้รับจาก Back-end ในหน้า Front-end อย่างถูกต้อง

6. นักเรียนสามารถจัดการกับข้อมูลจากฐานข้อมูลบนเซิร์ฟเวอร์ และแสดงผลให้ผู้ใช้เห็นในรูปแบบที่เข้าใจง่าย

1. นักเรียนสามารถใช้ HTTP methods (GET, POST, PUT, DELETE) ในการสื่อสารกับ API และรับส่งข้อมูลระหว่าง Front-end และ Back-end

2. นักเรียนสามารถสร้างและใช้ API เพื่อเชื่อมต่อระหว่าง Front-end และ Back-end ได้

3. นักเรียนสามารถทำงานกับ JSON หรือรูปแบบข้อมูลอื่นๆ ที่ถูกส่งผ่าน API ได้

4. นักเรียนสามารถใช้ JavaScript หรือเครื่องมืออื่นๆ ในการดึงข้อมูลจาก API และแสดงผลในหน้าเว็บ

5. นักเรียนสามารถตรวจสอบและแก้ไขข้อผิดพลาดที่เกิดขึ้นในการเชื่อมต่อ API และจัดการกับข้อมูลได้

6. นักเรียนสามารถวิเคราะห์ปัญหาด้านความปลอดภัย เช่น การจัดการกับข้อมูลที่รับมาจากผู้ใช้ และป้องกันการโจมตีจากภายนอก (เช่น Cross-Site Scripting หรือ Cross-Site Request Forgery)
